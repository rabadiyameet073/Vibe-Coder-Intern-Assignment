# Wobb Assignment: Influencer Search Application

## Repository
[https://github.com/Wobb-ai/vibe-coder-assignment](https://github.com/Wobb-ai/vibe-coder-assignment)

## Overview
Build on a starter influencer search application built with React, TypeScript, Vite, and Tailwind CSS.
The project is intentionally functional but not production-ready. Your goal is to improve the application by fixing issues, redesigning the UI, and enhancing the overall code quality.

---

## Getting Started
1. Clone or download the repository.
2. Install dependencies:
   ```bash
   npm install
   ```
3. Start the development server:
   ```bash
   npm run dev
   ```
4. Open your browser and visit:
   [http://localhost:5173](http://localhost:5173)

---

## What's Included
The starter project already includes:
- **Search / Dashboard**:
  - Filter influencers by platform (Instagram, YouTube, TikTok)
  - Search by username or full name
- **Profile Details**:
  - Click a profile to view detailed information loaded from JSON files
- **Routing**:
  - `/` — Dashboard
  - `/profile/:username` — Profile Details
- **Sample data**:
  - `src/assets/data/search/`
  - `src/assets/data/profiles/`

## Your Tasks

### 1. Find and Fix Issues
The project intentionally contains various bugs and code quality issues. Identify and resolve them while ensuring the application works correctly.

### 2. Redesign the UI/UX
Replace the existing interface with a modern and polished design. Focus on:
- Better user experience
- Responsive layout
- Good visual hierarchy
- Accessibility
- Consistent styling

### 3. Replace React Context with Zustand
The starter project uses React Context. Replace it with Zustand for state management.

### 4. Implement "Select Profile & Add to List"
The Add to List button is currently incomplete. Implement the complete feature with the following requirements:
- Add profiles to a selected list
- Prevent duplicate entries
- Display the selected profiles
- Allow removing profiles
- Keep the list persistent after page refresh

### 5. Improve Code Quality
Refactor the project where appropriate. Examples include:
- Better folder structure
- Cleaner components
- Reusable code
- Proper TypeScript types
- React best practices

### 6. Optimize Performance
Improve performance wherever appropriate. Examples:
- Avoid unnecessary re-renders
- Optimize component rendering
- Improve data loading
- Apply sensible React optimizations

### 7. Use Any Libraries
You are not restricted to the current stack. Feel free to use any libraries that improve your solution, including:
- UI libraries (e.g., Lucide React, Radix UI, Framer Motion)
- Animation libraries
- Utility libraries
- Testing libraries

- Data fetching libraries

Choose tools thoughtfully.
